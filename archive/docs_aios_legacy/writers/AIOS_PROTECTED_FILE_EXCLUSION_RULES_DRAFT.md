# AI_OS Protected File Exclusion Rules Draft

## Purpose

This draft defines protected-file exclusion rules for future writer planning.

No protected root files are edited by this draft. Human approval is required before any protected-file exception. This draft creates no live automation and no active writer.

- `README.md`
- `WHITEPAPER.md`
- `ARCHITECTURE.md`
- `RISK_POLICY.md`
- `DEPLOYMENT.md`
- `CHANGELOG.md`
- `AGENTS.md`
- `CLAUDE.md`
- `TODO.md`
- `REQUIREMENTS.md`
- `SOURCE_LOG.md`
- `ERROR_LOG.md`
- `HALLUCINATION_LOG.md`
- `DAILY_REPORT.md`
- `AAR.md`

## Exclusion Rule

Writers cannot edit protected files without explicit human approval, backup, validator pass, and commit checkpoint.

## Non-Approval Rule

This draft does not grant approval. Protected-file exceptions require a separate approval request naming the exact files, action, backup plan, expected result, and stop condition.

## Boundary

This draft does not activate writers, does not edit protected root files, and does not create live automation.
