# AI_OS Dashboard Operator Readability Accessibility Draft

## Purpose

This draft defines operator readability and accessibility expectations for future dashboard preview planning.

No protected root files are edited by this draft. Human approval is required before dashboard preview APPLY. This draft creates no live automation, no production dashboard, and no trading automation.

## Expectations

- High-contrast status labels.
- Plain language operator warnings.
- Clear PASS/REVIEW/NEEDS_REFACTOR/BLOCKED/INVALID DATA labels.
- Readable card layout.
- Keyboard navigation expectation.
- Visible stop conditions.
- Non-alarming but direct warning copy.
- Beginner-readable panel help.

## Failure Visibility Rule

The dashboard must not hide failures. Failed validators, blocked actions, unknown status, invalid data, and mismatches must be visible to the operator.

## Boundary

This draft does not activate a production dashboard, dashboard writer, live automation, or trading automation.
