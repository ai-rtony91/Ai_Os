# AI_OS Dashboard Validator Card Wiring Plan Draft

Status: DRAFT
Phase: Phase 12
Stage: 12.14 - Dashboard Status Implementation Readiness

## Purpose

Plan the validator health card for future dashboard implementation.

## Planned Inputs

- Overall validator health
- Validators discovered
- Validators run
- PASS count
- WARN count
- FAIL count
- SKIPPED count
- Latest health source

## Display Priority

FAIL and WARN states must appear before summary counts. Missing validator health must display UNKNOWN, not PASS.
