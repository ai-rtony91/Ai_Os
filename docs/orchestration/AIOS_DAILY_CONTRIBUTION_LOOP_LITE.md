# AIOS Daily Contribution Loop Lite

## Purpose

The daily contribution loop keeps AIOS work legitimate. It may surface small repo work only when there is no active higher-priority selected packet.

Green GitHub activity must be earned by useful validated repo work, not manufactured by noise.

## Good Daily Work

- doc polish that improves operator clarity
- test hardening
- repo hygiene documentation
- forex roadmap acceptance criteria
- paper-only evidence expansion
- dashboard simplification
- risk gate tests

## Forbidden

- fake commits
- fake heatmap commits
- whitespace-only churn
- generated noise
- generated Reports commits unless explicitly approved
- `control/review_bridge` commits unless explicitly approved
- broker, live, credential, order, webhook, scheduler, or daemon work

## Stop Point

The loop suggests one safe candidate and stops. It does not stage, commit, push, merge, dispatch workers, mutate queues, mutate approvals, or write reports by itself.
