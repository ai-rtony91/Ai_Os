# AI_OS Stage 102 Empty Stale Duplicate Doc Review Draft

## Purpose

This draft defines how AI_OS should detect empty, stale, duplicate, or under-evidenced documentation.

No protected root files are edited by this review. Human approval is required before correction or backfill. This draft creates no live automation, no backfill performed, no draft promoted, and no trading automation.

## Detection Targets

- Empty files.
- Tiny placeholder files.
- Stale files.
- Duplicate topic files.
- Conflicting instructions.
- Missing source evidence.
- Missing owner layer.
- Missing approval boundary.

## Labels

- PASS
- REVIEW
- NEEDS_REFACTOR
- BLOCKED
- INVALID DATA

## Reporting Rule

Review findings must be reported, not hidden. Conflicts with observed files, terminal output, screenshots, or known repo state must be marked REVIEW, BLOCKED, or INVALID DATA.

## Boundary

This draft does not edit protected root files, perform backfill, promote drafts, overwrite files, approve live automation, or touch broker/trading automation.
