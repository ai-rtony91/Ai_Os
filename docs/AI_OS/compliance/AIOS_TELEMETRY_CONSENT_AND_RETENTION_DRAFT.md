# AI_OS Telemetry Consent And Retention Draft

## Purpose

This draft defines telemetry consent and retention placeholders. It is documentation only and does not approve collection, storage, analytics providers, or telemetry writers.

## Consent Model Placeholder

Consent model is UNKNOWN. Future work must define opt-in, opt-out, consent logging, revocation, and operator review.

## Opt-In Telemetry Placeholder

Telemetry should be treated as opt-in until a future approved policy defines otherwise.

## Telemetry Classes

### User Telemetry

Future user telemetry may describe operator interactions, approvals, navigation, and help usage.

### App Telemetry

Future app telemetry may describe render state, panel status, validator visibility, and local dashboard health.

### Business Telemetry

Future business telemetry may describe plan tier, trial status, subscription status, and monetization planning fields.

## Blocked Data Classes

Blocked data includes credentials, secrets, API keys, broker tokens, private keys, recovery keys, payment card data, bank data, tax IDs, broker account data, private user material, live market execution data, and order path data.

## Retention Duration

Retention duration is UNKNOWN.

## Deletion / Export Process

Deletion and export processes are UNKNOWN.

## Non-Approval Statement

This draft does not approve telemetry collection, storage, analytics providers, telemetry writers, persistence, localStorage, sessionStorage, service workers, API calls, broker integration, or trading execution.
