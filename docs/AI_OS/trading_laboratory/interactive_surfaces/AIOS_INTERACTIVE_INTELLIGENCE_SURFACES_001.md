# AI_OS Interactive Intelligence Surfaces 001

Stage 16.3 turns visible Trading Lab intelligence into interactive explainable intelligence.

This layer remains paper-only. It does not add execution controls, broker controls, autonomous controls, hidden live routing, network order routing, installs, or secrets.

## Purpose

Interactive intelligence surfaces let the operator inspect why AI_OS trusts or distrusts conditions without exposing overwhelming clutter.

## Required Interactive Surfaces

- Confidence freeze timeline.
- Replay scrubber placeholder.
- Portfolio heat interaction.
- Edge decay inspection.
- Macro-risk overlay toggles.
- Survivability timeline.
- Freeze-trigger drilldown.
- Replay scenario switching.
- Risk escalation playback.
- Next Safe Action interaction surface.

## Visible States

- CONFIDENCE_ACTIVE
- CONFIDENCE_FROZEN
- WATCH
- UNSTABLE
- LOCKDOWN
- GOOD_EDGE
- WEAK_EDGE
- NO_EDGE
- ELEVATED_RISK
- PORTFOLIO_OVERLOAD

## Dashboard Behavior

- Preserve existing routing.
- Preserve paper-only safeguards.
- No autoplay.
- No execution controls.
- No broker controls.
- No hidden live routing.
- No clutter explosion.
- Interactive surfaces must remain explainable.
