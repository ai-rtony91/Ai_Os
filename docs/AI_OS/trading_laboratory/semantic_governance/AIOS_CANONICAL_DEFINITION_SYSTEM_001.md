# AI_OS Canonical Definition System 001

The canonical definition system is the governance memory for AI_OS trading language.

## Why It Exists

AI_OS needs stable definitions so modules do not interpret risk, confidence, edge, or recovery differently. Without a canonical index, one layer could treat a condition as safe while another treats it as unstable.

## Operating Role

The system provides machine-readable definitions for:

- DANGEROUS
- SAFE
- UNSTABLE
- STABLE
- GOOD_EDGE
- WEAK_EDGE
- NO_EDGE
- RECOVERY
- FAKE_RECOVERY
- LOCKDOWN
- PORTFOLIO_OVERLOAD
- CONFIDENCE_FREEZE

## Governance Rule

Definitions are anchors. They guide validation, scoring, and reasoning. They do not authorize execution.
