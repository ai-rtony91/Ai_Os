# AI_OS Execution Isolation Canon 001

Execution isolation is permanent.

AI_OS intelligence systems may recommend but not execute. No LLM belongs in a direct live order path. Any future execution system must be separate, audited, permissioned, and governed by controls outside this intelligence layer.

## Permanent Rules

- Intelligence may recommend but not execute.
- No LLM in direct live order path.
- Macro awareness modifies risk only.
- Execution isolation remains permanent.
- Replay testing required before trust increases.
- Edge validation required before confidence increases.

## Blocked States

- live_execution_status: BLOCKED
- broker_api_status: BLOCKED
- autonomous_execution_status: BLOCKED
- network_order_routing_status: BLOCKED

## Meaning

AI_OS can produce paper simulations, risk warnings, edge scores, confidence adjustments, and portfolio recommendations. Those outputs must not become broker orders without a separate future safety architecture.
