# AI_OS Screenshot Demo Capture Readiness Draft

## Purpose

This draft defines readiness rules for future screenshot and demo capture. It does not create screenshots, videos, or dashboard outputs.

Dashboard production outputs require separate approval.

## Capture Readiness Scope

Future capture readiness may prepare a checklist for approved static mock review, fixture-only capture, redaction, evidence notes, and operator training examples.

## Capture Non-Scope

There are no screenshots in this stage. There is no video in this stage. No app is launched, no browser is opened, and no production dashboard UI is created.

## Fixture-Only Capture

Future capture must use fixture-only data and approved mock state only. It must not use live telemetry, live trading data, broker screens, credentials, or uncontrolled screens.

## Redaction Requirements

Redaction is required before sharing evidence when paths, usernames, account identifiers, private details, or sensitive context are visible.

## Blocked Content

Future capture must include:

- no secrets
- no account IDs
- no broker data
- no uncontrolled screens
- no credentials
- no private keys
- no recovery keys
- no live trading data

## Future Stage 47

Future Stage 47 may define a screenshot checklist fixture. No capture artifact is created here.
