# AI_OS Dashboard Screenshot Demo Safety Rules Draft

## Purpose

This draft defines screenshot and demo safety rules for future dashboard preview evidence.

No protected root files are edited by this draft. Human approval is required before screenshot or demo capture workflows. This draft creates no live automation, no production dashboard, and no trading automation.

## Safety Rules

- Redact private paths where needed.
- No credentials.
- No tokens.
- No broker account data.
- No live market execution data.
- No browser profile details.
- No private user data.
- No upload of unsafe screenshots.
- Evidence naming convention.
- Demo-only fixture labeling.

## Capture Boundary

Screenshot/demo capture is not activated by this file. Future capture work requires explicit approval, safe fixture content, and validation before sharing or committing evidence.

## Boundary

This draft does not activate dashboard output, hidden capture behavior, live automation, or trading automation.
