# AI_OS Provider and Model Benchmark Boundary

## Status

Model/provider comparison is future-only. Phase 18.7 does not change providers, set up Azure, call OpenAI, call Azure OpenAI, call other providers, install packages, create `.env` files, or run network tests.

OpenAI vs Azure OpenAI and other providers may be benchmarked later only under a human-approved AI_OS packet.

## Future Benchmark Measures

Future benchmarks must measure:

- quality
- latency
- cost
- safety
- guardrail adherence
- packet correctness
- repo truthfulness
- profitability priority preservation

## Boundary Rules

- No current provider change.
- No Azure setup now.
- No API calls now.
- No real secrets.
- No real broker/API keys.
- No live trading tests.
- No Promptfoo install or execution in this phase.
- Future benchmark configs must not include secrets.
- Future benchmark targets must be AI_OS-owned fixtures or explicitly authorized local endpoints unless separately approved.

Trusted/proven profitability remains the top priority. Provider benchmarks must support AI_OS trusted/proven profitability by measuring safety, correctness, latency, and cost. They must not distract from Trading Lab / Forex evidence validation or create pressure toward unproven live execution.
