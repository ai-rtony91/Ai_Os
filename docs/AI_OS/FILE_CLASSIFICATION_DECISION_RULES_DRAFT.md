# AI_OS File Classification Decision Rules Draft

## Purpose

This draft defines file classification classes only. It does not approve cleanup, archive migration, file movement, deletion, renaming, overwrite, or protected-file edits.

## Classification Classes

| Class | Meaning | Allowed future action | Blocked action | Evidence before APPLY | Human approval required |
|---|---|---|---|---|---|
| STAY_ACTIVE | File belongs in the active AI_OS repo location. | Keep, document, reference, or index later. | Delete, move, rename, or overwrite without separate approval. | Exact path, purpose, and Git status. | YES for edits or structural changes. |
| PROTECTED_SOURCE_OF_TRUTH | File is a root or policy source-of-truth document. | Read, cite, or edit only after separate approval and backup/checkpoint. | Unapproved edit, overwrite, move, rename, delete, or auto-format. | Exact path, role, backup/checkpoint plan. | YES. |
| CREATE_INDEX_NOTE_LATER | Folder or area may need a new index/purpose note. | Create a new file only if target path does not exist. | Overwrite or replace an existing note. | `Test-Path` result showing target does not exist. | YES. |
| HUMAN_REVIEW_REQUIRED | Automated classification is not safe. | Read-only review and written recommendation. | Automated cleanup or classification finalization. | Evidence summary and unresolved question list. | YES. |
| ARCHIVE_CANDIDATE_LATER | File may belong in archive later. | Propose archive plan only. | Move, delete, rename, or archive without separate approval. | Replacement check, references check, and historical value review. | YES. |
| REPORTS_ONLY | File is a report, checkpoint, daily summary, or mismatch record. | Preserve and cite as historical evidence. | Rewrite, delete, move, or treat as source-of-truth code. | Report path, date, purpose, and references. | YES for edits. |
| INPUT_SOURCE | File is source input for docs, reports, or planning. | Preserve, cite, or review. | Convert, delete, or overwrite without separate approval. | Origin, content purpose, references. | YES. |
| INTERNAL_EVIDENCE | File is preserved evidence or source artifact. | Preserve, cite, or index later. | Modify, delete, rename, or move without separate approval. | Artifact path, source role, historical value. | YES. |
| GENERATED_OUTPUT | File was generated by a script, tool, or automation. | Preserve, regenerate later if approved, or summarize. | Treat as human source-of-truth without verification. | Generator/source, timestamp, command or workflow if known. | YES for replacement. |
| DUPLICATE_CANDIDATE | File appears similar to another file by name, path, or content. | Compare only and produce recommendation. | Delete, move, rename, overwrite, or merge automatically. | Exact paths, size, dates, Git status, references, content comparison. | YES. |
| LEGACY_REFERENCE | File is historical or legacy reference material. | Preserve or propose archive later. | Cleanup, delete, or modernize without approval. | Historical context and reference status. | YES. |
| INVALID_DATA | Claim conflicts with verified evidence or cannot be found where claimed. | Mark invalid in reports and request review. | Act on the claim as fact. | Terminal output, file listing, or screenshot/text evidence. | YES for logging/editing. |
| UNKNOWN | Fact or file role is not verified. | Investigate and label as UNKNOWN. | Assume, delete, move, rename, or overwrite. | Additional verified evidence required. | YES. |
