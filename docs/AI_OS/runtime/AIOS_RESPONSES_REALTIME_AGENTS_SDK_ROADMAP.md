Purpose:
Sequence future OpenAI-related runtime work without cluttering AI_OS or distracting from trusted/proven profitability.

## Current State

Current status: doctrine only.

AI_OS has local Phase 16, Phase 17, and Phase 18 scaffolds for bridge planning, execution pipeline preview, and improvement loop preview. Real OpenAI API use remains blocked until a separate human-approved APPLY packet.

## Roadmap

1. Responses API boundary

2. Local adapter fixture
   Extend local fixtures before any real API call. Keep no-write validation mode.

3. Realtime API boundary for Pi car voice
   Define voice session ownership, audio safety, no motor-control rule, and session stop behavior.

4. Agents SDK orchestration
   Introduce only after manager/tool/handoff rules are validated. AI_OS Manager remains default owner.

5. ChatKit
   Consider later for UI or chat embed needs. Not a current execution priority.

6. Agent Builder
   Consider later for visual workflow design or reference. Not required now.

## North Star

Trusted, proven profitability remains the North Star. Runtime expansion must support evidence quality, validator trust, operator workload reduction, and safe paper-only Trading Lab progress.
