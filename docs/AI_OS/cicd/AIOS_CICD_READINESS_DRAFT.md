# AIOS CI/CD Readiness Draft

Status: Draft scaffold

## Purpose

Define readiness requirements before AI_OS can use CI/CD workflows.

## Readiness Checks

- Repository health validation.
- Test command inventory.
- Lint and formatting expectations.
- Build artifact policy.
- Release checkpoint requirement.
- Rollback plan requirement.
- Secret exclusion validation.

## Blocked Actions

- Creating production deployment workflows without approval.
- Adding credentials.
- Auto-deploying to cloud.
- Triggering live broker or trading paths.
