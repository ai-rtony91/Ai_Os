# AI_OS Screen Recording Evidence Policy Draft

## Purpose

This draft defines future evidence rules for optional screen recording. It does not approve recording automation.

## Draft Policy Rules

- No silent recording is allowed.
- User approval is required before every recording.
- Microphone must default to OFF.
- Codex opening may mark a work-session start, but must not start recording automatically.
- No auto-recording from Codex is allowed until separately approved.
- Recording must stop before passwords, broker pages, email, bank pages, private keys, API keys, tokens, recovery keys, or personal documents are shown.
- Recordings may be stored only in a future approved evidence folder.
- A retention and delete policy is required before any recording automation.
- Recordings must be treated as sensitive evidence until reviewed.
- Text reports and telemetry are preferred when they can capture the needed evidence safely.

## Sensitive-Window Stop Rules

Stop recording before opening or exposing:

- Password managers or password prompts.
- Broker, trading, or financial pages.
- Email or private messaging.
- Banking, payment, or tax records.
- API keys, tokens, private keys, recovery keys, or credentials.
- Personal documents, IDs, addresses, phone numbers, or private account pages.

## Automation Limit

This draft does not authorize screen recording, screenshots, recording tool setup, retention automation, file movement, or evidence-folder creation.
