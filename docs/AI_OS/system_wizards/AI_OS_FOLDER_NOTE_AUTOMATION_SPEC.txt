// AUTOMATION GOAL
Create a safe automation script that adds one README_FOLDER_PURPOSE.txt file into approved folders.

Each README_FOLDER_PURPOSE.txt explains:
- what the folder is for
- what belongs inside
- what does not belong inside
- owner/context
- risk level
- whether the folder is AI_OS, TRADING_SYSTEM, REPORTING, SECURITY, BACKUP, or REVIEW
- whether automation may write to the folder


// REQUIRED MODE DESIGN
The automation must support two modes:

1. DRY_RUN
   - scans folders
   - shows what it would create
   - shows what it would skip
   - writes a report
   - makes no changes

2. APPLY
   - creates README_FOLDER_PURPOSE.txt only after approval
   - does not overwrite existing README_FOLDER_PURPOSE.txt files
   - writes a final report
   - opens report in Notepad


// ABSOLUTE SAFETY RULES
1. Never delete files.
2. Never move files.
3. Never rename files.
4. Never overwrite existing files.
5. Never edit README.md, AGENTS.md, RISK_POLICY.md, SOURCE_LOG.md, ERROR_LOG.md, HALLUCINATION_LOG.md, AAR.md, DAILY_REPORT.md, or White_Paper.md without backup.
6. Never touch BitLocker, BIOS, registry, firewall, VPN, browser policies, broker API keys, or live trading settings.
7. Never assume a folder purpose without labeling it REVIEW_NEEDED when unclear.
8. Never accept inventory reports that conflict with File Explorer, screenshots, terminal output, or known folder state.
9. If PowerShell shows parser errors, continuation prompts, or impossible counts, stop and inspect.
10. One script at a time.
