# 03_ARCHITECTURE_LOCK

## Locked Rule (Authoritative)
AI_OS is OMEN-first. The OMEN desktop is the local command/control foundation. Codex/GitHub is the implementation and repository layer. AI tools operate underneath the system-level AI_OS architecture, not above it. ChatGPT is architect/planner/documentation controller. Codex is repository/code implementer. Claude Code is secondary code reviewer/surgeon. GitHub is source control. The user remains execution authority. This structure is locked unless a newer development method clearly improves the build.

## Interpretation
- Local-first control stays on the OMEN system.
- Cloud/repo tooling supports implementation, not governance override.
- Human approval authority remains with the user.

## Change Threshold
Architecture changes require:
1. Clear evidence of improvement
2. Explicit comparison versus current method
3. Documented migration notes
4. User approval before adoption

## Non-Negotiables
- Do not invert role hierarchy.
- Do not promote tools above system architecture policy.
- Do not bypass user execution authority.
