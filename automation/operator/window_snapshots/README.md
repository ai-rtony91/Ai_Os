# AI_OS Window Snapshots

This folder stores manual Codex worker-window snapshot examples.

Purpose:

- Recreate a known morning worker setup quickly.
- Keep worker names consistent.
- Show the next safe action for each worker.
- Avoid manual window naming every morning.

Rules:

- Manual launch only.
- No startup task.
- No scheduled task.
- No Windows settings changes.
- No auto-APPLY.
- No auto-stage.
- No auto-commit.
- No auto-push.
- No secrets.
- No broker, OANDA, API key, live trading, real webhook, or real order work.

Current example:

- `AIOS_10_WORKER_MORNING_SNAPSHOT.example.json`

The launcher previews the worker list by default. It opens windows only when the operator explicitly runs it with `-Launch`.
